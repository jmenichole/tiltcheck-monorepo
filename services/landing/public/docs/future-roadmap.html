15. Future Roadmap

This roadmap outlines the phased evolution of the TiltCheck ecosystem.

Timelines are flexible — built for a solo founder with limited funding, meaning progress follows opportunity, grants, and available resources.

All features are modular, optional, and can be released independently.

---

15.1 Philosophy of Development

TiltCheck evolves according to 3 principles:

1. Ship the most helpful modules first

Anything that reduces chaos, scams, or degen confusion gets priority.

2. Avoid over-building

No big bangs.

No massive frontend builds.

Discord-first until scaling demands a UI.

3. Optimize for free-tier resources

Cloudflare

Supabase

GitHub Actions

Magic

Open libraries

Jupiter

Public APIs

TiltCheck’s roadmap assumes scrappy resourcefulness.

---

15.2 Phase 1 — Core Launch (MVP)

Modules included:

  • JustTheTip (non-custodial tipping)
  • SusLink (URL scanner)
  • CollectClock (bonus logging + nerf detection)
  • FreeSpinScan (promo submission + classification)
  • Base casino trust engine
  • Basic degen trust score
  • TiltCheck Core (tilt nudges + cooldown commands)
  • NFT identity (optional, minimal)
  • Agent-ready architecture

Discord-first release:

Focus on:

  • stable commands
  • reliable scanning
  • simple tipping
  • smooth mod workflow

Outcomes:

  • community stability
  • reduction in scam links
  • cleaner promo channels
  • safer tipping
  • early trust signals

15.3 Phase 2 — Intelligence Expansion

Additions:

  • FreeSpinScan prediction engine
  • CollectClock bonus cycle prediction

  • Advanced degen trust scorer

Outcomes:

  • survey revenue potential opens
